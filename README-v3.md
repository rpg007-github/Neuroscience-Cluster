## T4 Migration Guide for UWF Webmaster

This prototype is structured to make your T4 build as direct as possible.

### What T4 Provides Automatically (Remove from Prototype)

These are loaded globally by T4 and should **not** be added manually:
- `<link>` to `/media/uwf-mstoner/site-assets/css/main.css` (Bootstrap 4 + all UWF classes)
- Google Fonts: Montserrat (already in main.css)
- Font Awesome kit `a098e44fc9`
- Google Tag Manager (GTM-P9W355)
- UserWay accessibility widget
- BrightEdge Autopilot
- Slate Ping script
- UWF global header (`<header class="bg-primary header">`)
- UWF global footer (`<footer class="site-footer">`)
- Bootstrap 4 JS + jQuery slim

### What You Build in T4

Replace each section in `index.html` with the corresponding T4 content type:

| Section | T4 Content Type |
|---------|----------------|
| Hero jumbotron | Jumbotron Image content type |
| College nav | `nav-college` (auto-generated by T4 for your section) |
| Mission lead text | Standard page content with `.lead` class |
| Research cards (blue panel) | Panel content type → `panel--blue` |
| Team bios (alternating) | Full-Width Columns content type → `alternating` rows |
| Publications | Standard content with custom HTML (submit JS filter for UMC vetting) |
| Grants (blue section) | Panel content type → `bg--uwfblue` |
| NeuroNauts (alternating) | Full-Width Columns content type |
| Contact (grid) | Standard content with `grid--slider` row |
| Nautilus footer block | Nautilus content type (already in T4 as `bg-nautilus`) |
| Contact form | Wufoo embed (per UWF policy — contact websupport@uwf.edu) |

### Faculty Headshots

Replace the two-letter avatar initials (`.neuro-avatar` elements) with actual headshots from the UWF Photography Library (`uwfphotos.smugmug.com`) or commission through UWF Creative Services. Per T4 pattern, use `<img>` inside the alternating column image div.

### Events Feed

Add a UWF Event Calendar RSS feed to pull cluster events automatically. Contact websupport@uwf.edu for setup. This prevents stale event content (required by UWF content review policy).

### Publications (Long-Term)

The manually maintained publication list can eventually be replaced with a feed from UWF's institutional repository at `ircommons.uwf.edu` (BePress/Esploro). Contact the library for integration options.

## GitHub Pages Deployment (Prototype Only)

1. Create a GitHub repository
2. Upload `index.html`, `custom-neuro.css`, `app-neuro.js`
3. Go to **Settings > Pages > Source: main branch / root**
4. Live at `https://[username].github.io/[repo-name]/`

Note: GitHub Pages prototype uses CDN Bootstrap 4 and Font Awesome. These are replaced by T4's `main.css` in production.
